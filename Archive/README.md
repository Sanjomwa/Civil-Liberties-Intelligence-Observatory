# Archive

This folder holds superseded documents — archived, not deleted, so the history of how CLIO's documentation evolved stays inspectable. Nothing here should be treated as a current source of truth; each entry below states what superseded it.

`Archive/*.docx` files are gitignored (internal strategy/pricing material, kept out of the public repo deliberately — see `.gitignore`); the `.md` files below are plain historical documentation and are tracked normally.

## 2026-07-12 — seven pre-restructure `docs/` root files (TD-23, TD-24)

These predate the `docs/00-07` numbered structure adopted after the Project Zero Review, and describe an earlier state of the project (a Kimball/CLRI-style dimensional model, mock Lumen data instead of the current synthetic-but-pipeline-integrated version, a Cloud Run deployment target that was never built, dashboard names that don't match the current 9 Streamlit pages). None were deleted outright — moved here per this project's own archive-not-delete convention.

- **`readme1_1.md`** — an earlier, more elaborate README draft. TD-24's disposition ("reconcile the ethics section back into the current README, then archive the draft") was executed: its Ethics and Responsible Use content was reviewed against the current `README.md`, three genuinely missing safeguards were folded in (explicit no-PII/no-deanonymization commitment, a no-exploitation-tooling harm-minimization statement, and an explicit "don't use outputs to justify harm or discrimination" clause), and the rest of the draft (architecture diagrams, dashboard descriptions, setup instructions) is fully superseded by the current `README.md` and does not need reconciling.
- **`data-modelling.md`**, **`data_sources.md`**, **`erd-lineage.md`** — TD-23 found these describe an architecture (a `civil_liberties_mart` unified mart, `dim_event_type`/`dim_platforms`/`dim_reasons` as live conformed dimensions, `fact_censorship_measurements`/`fact_takedown_requests`) that doesn't match the actual implemented reporting layer. Superseded by freshly written versions grounded in the live repo and BigQuery schema: [`docs/02-architecture/data-modelling.md`](../docs/02-architecture/data-modelling.md), [`docs/02-architecture/data_sources.md`](../docs/02-architecture/data_sources.md), [`docs/02-architecture/erd-lineage.md`](../docs/02-architecture/erd-lineage.md).
- **`analysts-questions-playbook.md`**, **`civil-liberties-reporting-playbook-Kenya.md`** — pre-restructure analyst-facing docs referencing the same superseded schema vocabulary as the three files above (`civil_liberties_mart`, mock Lumen data). No direct replacement was written in this pass; the current `docs/02-architecture/data_sources.md` and the Streamlit dashboard's own per-page descriptions (README's "Dashboard Showcase") cover the same ground at the current architecture's vocabulary.
- **`project_difficulties.md`** — a retrospective on early project difficulties. Note: `docs/03-development/documentation-standards.md` previously pointed at this file's original `docs/` root location as "the template to formalize going forward" for future retrospectives; that reference has been updated to point here. The file's content is historical (describes difficulties from the project's pre-restructure phase) and is preserved as the template TD-24's own documentation-standards entry describes, not treated as still-current operational guidance.

See `docs/02-architecture/technical-debt-inventory.md` (TD-23, TD-24 — both closed) and `docs/02-architecture/decision-log.md`'s 2026-07-12 entry for the full reasoning behind this disposition.
